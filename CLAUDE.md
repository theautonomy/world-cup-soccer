# World Cup Soccer — 2026 FIFA World Cup Agent Skill

## What this is

An AI agent skill that researches and generates structured JSON data for all 48 nations competing in the 2026 FIFA World Cup. A React app (deployed via GitHub Actions to GitHub Pages) renders the data into bilingual (Simplified Chinese / English) team profile pages.

Invoke with: `/world-cup team {name}`

## Project Structure

```
world-cup-soccer/
  .claude/skills/world-cup/SKILL.md   — skill router
  modes/team.md                        — team data mode (research + JSON output)
  data/teams/                          — generated JSON files (committed, source of truth)
    {slug}.json                        — one file per team
    index.json                         — auto-generated by CI, lists all teams
  app/                                 — React + Vite app (renders JSON)
    src/
      components/                      — reusable bilingual components
      pages/                           — Home (team grid) + TeamPage (full profile)
      context/AppContext.jsx           — lang + theme state (localStorage-backed)
      styles/global.css                — CSS variables, theme, typography
    public/data/teams/                 — JSON copied here by CI before build
  dev.mjs                              — local dev script (sync JSON + run app)
  templates/team-template.html        — legacy HTML template (kept for reference)
  output/teams/                        — legacy HTML output (gitignored)
  .github/workflows/deploy-pages.yml  — CI: copy JSON → app/public → npm build → deploy
  CLAUDE.md                            — this file
```

## How team mode works

1. **Check** — does `data/teams/{slug}.json` exist?
   - No → full research → write JSON
   - Yes → read existing → identify gaps → targeted research → update JSON
   - `--new` flag → force full regeneration
2. **Research** — WebSearch for country info, WC history, 2026 squad, legends, rivalries, fun facts
3. **Write** — save to `data/teams/{slug}.json`
4. **Test locally** — `node dev.mjs` syncs JSON and starts the dev server
5. **Deploy** — push to `master` triggers GitHub Actions: copies JSON → builds React → deploys to GitHub Pages

## JSON Schema

Full schema is in `modes/team.md`. Key fields:

```json
{
  "meta": { "slug", "generated", "version" },
  "team": { "name_zh", "name_en", "flag_emoji", "federation", "fifa_rank", "colors" },
  "country": { capital, population, area, language, continent, currency, anthem, desc, football_desc },
  "world_cup": { titles, appearances, best_result, title_years, history[] },
  "squad_2026": { coach, players[] },
  "legends": [ { name, years, position, bio, stat } ],
  "rivalries": [ { opponent, desc } ],
  "fun_facts": [ { icon, text } ]
}
```

## React App Architecture

- **Framework**: React 18 + Vite 5 + React Router v6
- **Base URL**: `/world-cup-soccer/` (GitHub Pages subdirectory)
- **Language toggle**: `AppContext.lang` → `data-lang` on `<html>` → CSS shows/hides `.zh`/`.en` spans
- **Theme toggle**: `AppContext.theme` → `data-theme` on `<html>` → CSS variable overrides
- **Team colors**: `TeamPage` sets `--color-primary/secondary/accent` via `document.documentElement.style`
- **Data loading**: `fetch('/world-cup-soccer/data/teams/{slug}.json')` at runtime
- **Team index**: `fetch('/world-cup-soccer/data/teams/index.json')` — built by CI

## Bilingual Implementation

Every text element uses the `<T>` component:
```jsx
<T zh="中文内容" en="English content" />
```

`T` reads `lang` from `AppContext` and renders the matching string.

## Output Naming

`data/teams/{slug}.json` where slug = lowercase English team name, spaces → hyphens.

Examples: `brazil.json`, `south-korea.json`, `united-states.json`

## Local Development

```bash
node dev.mjs        # sync JSON + start Vite dev server  → http://localhost:5173/world-cup-soccer/
node dev.mjs prod   # sync JSON + production build + preview → http://localhost:4173/world-cup-soccer/
```

`dev.mjs` does three things:
1. Copies `data/teams/*.json` → `app/public/data/teams/`
2. Generates `app/public/data/teams/index.json` from all present team files
3. Starts the React dev server (or builds + starts preview for `prod`)

Re-run after generating any new team JSON to pick it up.

## Adding New Modes

1. Create `modes/{mode}.md` with research and generation instructions
2. Add routing entry to `.claude/skills/world-cup/SKILL.md`
3. Update the modes table above
