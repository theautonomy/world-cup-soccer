# Mode: refine — 球队数据刷新

读取现有 JSON，聚焦于**时效性**内容，搜索最新资讯并定向更新。与 `team` 模式的区别：`team` 补充缺失内容；`refine` 专注刷新已有内容的准确性与时效性。

## 调用方式

```
/world-cup refine Brazil
/world-cup refine 法国
```

---

## 工作流

```
1. 确定 slug → 读取 data/teams/{slug}.json
2. 运行审计：node audit.mjs {slug}
3. 识别刷新目标（见下方优先级）
4. 定向搜索最新资讯
5. 用 Edit 工具更新字段
6. 更新 meta.generated 为今日日期
```

如果文件不存在 → 提示用户先运行 `/world-cup team {name}`。

---

## 刷新优先级

### 高优先级（必搜）

| 字段 | 搜索目标 |
|------|---------|
| `squad_2026.coach` | 主教练是否更换？合同状态？ |
| `squad_2026.players` | 最新征召名单、球员伤病、退队、新面孔 |
| `team.fifa_rank` | 最新 FIFA 排名（每月更新） |
| `world_cup.history` | 是否有最新赛事结果需补充 |

### 中优先级（按需搜）

| 字段 | 搜索目标 |
|------|---------|
| `players[*].caps` | 出场数是否有变化 |
| `players[*].club` | 转会动态（冬季/夏季窗口） |
| `fun_facts` | 是否有新的有趣数据值得补充 |

### 特殊规则：`world_cup.poem_zh`（必检）

读取 JSON 后，**立即检查 `world_cup.poem_zh` 是否为空字符串**。若为空，**必须生成七律诗**再进行其他刷新。

**七律诗写作规范：**
- 格式：8行，每行7个汉字，押平声韵（偶数行末字同韵：第2、4、6、8行）
- 主题：用诗意语言回顾该队世界杯历史，兼顾荣耀与遗憾，末联展望2026
- 典故：融入该队标志性时刻（夺冠年份、经典比赛、传奇球员、历史低谷）
- 风格：庄重典雅，不用现代口语；地名人名可用音译简称（如「贝利」「米内」「伯尔尼」）

**示例（巴西）：**
```
五捧大力震寰宇，
桑巴热浪席全球。
贝利十七惊世界，
罗纳二度铸金瓯。
马拉卡纳留旧恨，
米内罗场七球愁。
美丽足球今未老，
二六征途再扬舟。
```

生成后用 Edit 工具写入 `world_cup.poem_zh`。

### 低优先级（仅在审计警告时处理）

| 字段 | 处理方式 |
|------|---------|
| 叙述文字过短 | 补充扩写 |
| 传奇球员介绍简略 | 补充细节 |
| 宿敌描述薄弱 | 补充历史背景 |

## 推荐搜索策略

```
"{team name} 2026 World Cup squad"           → 最新阵容
"{team name} coach 2025 2026"                → 教练动态
"{team name} FIFA ranking 2025"              → 最新排名
"{team name} injury news 2026"               → 伤病情况
"{team name} transfer news 2025"             → 转会动态
"FIFA World Cup 2026 {team name} squad list" → 官方名单
```

**来源优先级：** FIFA.com → transfermarkt.com → BBC Sport → ESPN → goal.com

---

## 更新规则

- **只更新有据可查的字段** — 搜索结果不明确时保留原值
- **球员出场数** — 只增不减（历史出场数不会倒退）
- **俱乐部** — 若发现转会则更新，并在球员条目旁备注转会时间
- **教练变更** — 更新 `coach` 对象，可在 `fun_facts` 中加一条关于新教练的信息
- **FIFA 排名** — 直接覆盖 `team.fifa_rank`
- **新球员** — 添加到 `players` 数组；已退出国家队的球员标注 `"caps"` 但不删除（历史记录）

## 输出步骤

1. 列出本次刷新了哪些字段（变更摘要）
2. 列出哪些字段因信息不足未作更改
3. 如果审计发现内容问题（字段过短等），列出建议但不强制修改
4. 报告更新后的文件路径和 `meta.generated` 日期
